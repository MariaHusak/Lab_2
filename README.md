# Paw Runner

**Paw Runner** — 2D платформер з керованим персонажем-собакою, що стрибає через перешкоди.  
Проект розробляється в рамках лабораторної роботи з використанням каскадної моделі життєвого циклу ПЗ (**Waterfall**).

---

## 1. Оцінка та Планування (Requirements)

**Детальний аналіз етапу:**  
На цій стадії визначено основні механіки гри та межі проекту. Обрана модель Waterfall передбачає, що будь-які зміни після завершення цього етапу є небажаними та дорогими.

**Вимоги:**

**Функціональні:**
- Гра запускається як GUI-додаток (PySide6).
- Керування гравцем: `Space` — стрибок, утримання клавіші для довшого стрибка; `R` — перезапуск після Game Over.
- Автоматична генерація перешкод з випадковим інтервалом.
- Підрахунок та відображення рахунку (score) у верхньому лівому куті.
- Game Over при зіткненні з перешкодою.

**Нефункціональні:**
- Тип додатку — GUI.
- Мова — Python 3.14.
- Плавність анімації ≥30 FPS.
- Візуальний стиль — простий 2D-платформер із прямокутними об’єктами та базовою анімацією.

**Планування:**
- **Обсяг робіт:** Реалізація класу гри, таймера, логіки стрибка, генерації та відображення перешкод, колізій, рахунку, Game Over, тестів.
- **Ресурси:** 1 розробник, PyCharm, PySide6, стандартні бібліотеки Python.
- **Часові рамки:** Розробка — 6 годин, тестування — 2 години, документування — 1 година.

## 2. Проектування (Design)

**Обрано об’єктно-орієнтований підхід** із чітким розділенням відповідальності (Single Responsibility Principle).

**Архітектура:**

**Platformer (QWidget)**  
- **Відповідальність:** Основний клас гри, управління фізикою гравця, генерацією перешкод, колізіями, рахунком та відображенням.  
- **Методи:**  
  - `reset_game()` — ініціалізація стану гри.  
  - `spawn_obstacle()` — створення нових перешкод.  
  - `keyPressEvent()` / `keyReleaseEvent()` — обробка натискання клавіш.  
  - `game_loop()` — оновлення стану гри (фізика, колізії, рахунок).  
  - `paintEvent()` — малювання сцени.  
  - `draw_dog()` / `draw_obstacle()` — малювання гравця та перешкод.  
- **Атрибути:** стан гравця, швидкість по Y, список перешкод, рахунок, стан Game Over.

---

### 2.2 Потік даних (Data Flow)

1. Користувач запускає `main.py`.
2. Створюється QApplication та об’єкт Platformer.
3. Platformer запускає таймер, який кожні 30 мс викликає `game_loop()`.
4. Клавіші змінюють стан гравця: стрибок, утримання, перезапуск.
5. `game_loop()` обчислює фізику, оновлює позиції перешкод, перевіряє колізії.
6. `paintEvent()` малює оновлену сцену.
7. Рахунок та статус гри відображаються у верхньому лівому куті.

**Дизайн інтерфейсу:**  
- Простий 2D-графічний інтерфейс.  
- Колірне виділення: гравець — коричневий, перешкоди — зелені та коричневі блоки, фон — темно-синій.  
- Відображення рахунку та Game Over текстом.

## 3. Реалізація (Implementation)

**Детальний аналіз етапу:**  
Код реалізовано суворо відповідно до проектування. Фізика стрибка оптимізована через комбінацію гравітації та сили утримання клавіші. Перешкоди генеруються динамічно та видаляються після виходу за межі екрану.

**Вибір технологій:**
- Python 3.14  
- PySide6 (QWidget, QPainter, QTimer)  
- Стандартні бібліотеки: `random`, `sys`

**Програмування:**  
- Клас Platformer реалізує весь ігровий цикл та GUI.  
- Таймер оновлює стан гри та викликає `update()`.  
- Колізії перевіряються через пересічення `QRect` гравця та перешкод.  
- Підрахунок рахунку кожним циклом.  
- Game Over блокує фізику та відображає інструкцію до перезапуску.

---

## 4. Тестування (Testing)

**Детальний аналіз етапу:**  
Тестування виконано після завершення розробки, згідно з моделлю Waterfall. Використано `pytest` для модульного тестування.

**План тестування (Test Suites):**
- **TestInitialState:** перевірка початкового стану гравця та перешкод.  
- **TestObstacle:** перевірка генерації та видалення перешкод.  
- **TestPlayerPhysics:** перевірка стрибка, утримання клавіші, гравітації та приземлення.  
- **TestCollision:** перевірка Game Over при зіткненні.  
- **TestScore:** перевірка збільшення рахунку.  
- **TestRestart:** перевірка перезапуску гри після Game Over.

## 5. Впровадження (Deployment)

**Реліз продукту:**  
Гра готова до запуску на локальних машинах із Python 3.14 та PySide6.

**Інструкція з розгортання:**
1. Клонувати репозиторій на локальну машину.
2. Встановити Python 3.14 та PySide6:
   ```pip install PySide6```

**Запуск гри:** 
```python main.py```

**Управління:**
Space — стрибок (утримання для довшого стрибка)
R — перезапуск після Game Over
